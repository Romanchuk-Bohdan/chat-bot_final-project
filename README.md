# Smart Habit & Finance Tracker

Десктопний застосунок на основі **WPF (.NET 8)**, призначений для обліку фінансів (доходи/витрати), відстеження звичок та постановки фінансових цілей. Проєкт реалізований з дотриманням принципів чистої архітектури та сучасних патернів проєктування.

---

## Функціонал

* **Керування користувачами**: Створення, перейменування, перемикання профілів.
* **Фінанси**: Облік витрат та доходів.
* **Фінансові цілі**: Створення цілей накопичення, додавання коштів, відстеження прогресу.
* **Трекер звичок**: Створення звичок, відстеження виконання, серії (streaks).
* **Аналітика**: Статистика фінансових потоків.
* **Звіти**: Генерація звітів у форматах `.txt` та `.csv` через патерн **Factory Method**.

---

## Архітектура (N-Tier)

Проєкт розділений на логічні шари для забезпечення ізоляції та чистоти коду:

1.  **`ChatBot.Core`**: Моделі даних та інтерфейси (контракти).
2.  **`ChatBot.Data`**: Реалізація репозиторіїв, збереження даних у форматі JSON.
3.  **`ChatBot.Services`**: Бізнес-логіка, рушій команд (`ChatEngine`), обробка стратегій.
4.  **`ChatBot.UI`**: Графічний інтерфейс (WPF) на основі патерну MVVM.

## Programming Principles

Згідно з вимогами до проєкту, реалізовано 5 ключових принципів:

* **SRP (Single Responsibility Principle)**: Кожен клас виконує одну задачу (наприклад, `AddExpenseCommand`).
* **OCP (Open/Closed Principle)**: Система відкрита для розширення (додавання нових команд), але закрита для модифікації `ChatEngine`.
* **DIP (Dependency Inversion Principle)**: Модулі залежать від абстракцій (`IUserRepository`), а не від конкретних реалізацій.
* **KISS (Keep It Simple, Stupid)**: Використання зрозумілих слеш-команд для взаємодії.
* **DRY (Don't Repeat Yourself)**: Централізована обробка команд та використання репозиторіїв.

## Refactoring Techniques

Використані техніки для покращення коду:

1.  **Extract Class**: Розбиття загального класу керування ботом на окремі класи-команди.
2.  **Replace Conditional with Polymorphism**: Заміна великих `switch-case` конструкцій у `ChatEngine` на поліморфний виклик `Execute()`.
3.  **Rename Method/Variable**: Використання змістовних назв (наприклад, `Category.cs` був перейменований в `Expense.cs` коли стара назва стала не актувальною).
4.  **Dead Code Elimination**: Видалення застарілих проєктів та невикористовуваних папок.
5.  **Parameterize Method**: Використання інтерфейсів у конструкторах для впровадження залежностей (DI).

## Інструкція із запуску

1.  Клонуйте репозиторій.
2.  Відкрийте файл `ChatBot.sln` у **JetBrains Rider** або **Visual Studio 2022**.
3.  Переконайтеся, що встановлено **.NET 8 SDK**.
4.  Зберіть проєкт (**Build Solution**).
5.  Запустіть проєкт `ChatBot.UI`.
6.  У вікні чату введіть `/help` для ознайомлення з доступними командами.

> **Примітка**: Загальна кількість рядків коду (.cs): ~2201.
